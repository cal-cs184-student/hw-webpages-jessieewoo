Task 1: Drawing Single-Color Triangles

Triangle Rasterization

To rasterize a triangle, I implemented a helper function that performs a 2D cross-product sign test. For each edge of the triangle and a sample point, I compute: \[ (x - x_a)(y_b - y_a) - (y - y_a)(x_b - x_a) \] This determines whether the point lies to the left or right of the directed edge. To test whether a point lies inside the triangle: - I compute the signed area test for all three edges. - If all signs are consistent (all non-negative or all non-positive), the point lies inside. - Otherwise, it lies outside. I compute the triangle’s bounding box using min/max of vertex coordinates, convert to integer pixel indices with floor/ceil, then iterate over each pixel in the box. I sample at pixel centers (x + 0.5, y + 0.5). If inside, I fill the pixel.

Runtime Analysis

My algorithm iterates only over the bounding box and performs constant work per sample (three edge tests). Therefore runtime is: \[ O(B) \] where B is the number of samples in the bounding box. This is asymptotically no worse than checking each sample in the bounding box.

Task 2: Antialiasing by Supersampling

Algorithm

Instead of storing one color per pixel, I implemented a sample_buffer storing multiple subsamples per pixel. - Let sample_rate = N². - Each pixel becomes an N × N grid. - For each pixel in bounding box: - Loop over all subsamples. - Compute subsample position: \[ (x + \frac{i+0.5}{N}, \; y + \frac{j+0.5}{N}) \] - Perform triangle edge tests. - Store subsample color. After rasterization, I average subsamples in resolve_to_framebuffer.

Why Supersampling Works

Sample rate 1 → binary coverage → jagged edges. Higher sample rates → fractional coverage → smooth blending.

Task 4: Barycentric Coordinates

Barycentric coordinates represent any point inside a triangle as weighted sums of its vertices: \[ P = \alpha A + \beta B + \gamma C \] where: \[ \alpha + \beta + \gamma = 1 \] If a point is closer to a vertex, that vertex’s weight is larger. This allows smooth interpolation of attributes such as color: - Red vertex → high red weight - Green vertex → high green weight - Blue vertex → high blue weight This produces smooth gradients. ---

Task 5: Pixel Sampling for Texture Mapping

Pixel Sampling

For each pixel: - Compute interpolated (u, v). - Convert to texture space. - Sample texel.

Nearest Sampling

- Round to nearest texel. - Fast but blocky.

Bilinear Sampling

- Sample four neighboring texels. - Interpolate horizontally and vertically. - Produces smoother results.

Task 6: Level Sampling with Mipmaps

Level Sampling

A mipmap is a pyramid of textures at different resolutions. To compute mip level: - Measure change in UV across screen space. - Convert to texel space. - Compute footprint. - Use: \[ \text{level} = \log_2(\text{footprint}) \] Modes: - L_ZERO → always level 0 - L_NEAREST → nearest mip - L_LINEAR → interpolate between two mips

Tradeoffs

  • Pixel Sampling: Nearest fast, bilinear smoother.
  • Level Sampling: Reduces aliasing when zoomed out.
  • Supersampling: Smooths triangle edges but increases memory and runtime.
Highest quality: - P_LINEAR - L_LINEAR - High sample rate Highest cost: - More computation and memory.
